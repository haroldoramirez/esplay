Engenharia de Software– Sistema Esplay

1 Introdução

Este documento tem por objetivo servir como um manual de instruções, de forma a dar fundamentos para o usuário operar o sistema Esplay. Além disso, demonstrar as características do sistema, bem como suas telas e funcionalidades.

2 Sistema Web

O objetivo deste produto é ajudar as pessoas gerenciar seu compromissos de maneira prática. Trata-se de um sistema simples e fácil de usar. O sistema conta com a opção de gerenciar usuários, agendar compromissos, criar categorias de compromissos, criar tipos de compromissos, gerenciar contatos e mostrar os logs que acontecem no sistema.

3.4 Agendar Compromisso

Para agendar um novo compromisso o Usuário deve escolher a opção "Agenda" na tela inicial do sistema e então no botão "Novo Compromisso". O Sistema vai exibir a seguinte tela, para preenchimento das informações sobre o projeto:

Ilustração 4: Tela de Agendar Compromissos


A descrição, observação e o local do compromisso são os únicos campo que não precisam ser preenchidos obrigatoriamente. O campo "Data" deve ser preenchido apenas selecionando a opção através do calendário. Para preenchimento da hora, deve-se selecionar os valores com as flechas para cima e para baixo ou (apenas números). O "status", tem apenas duas opções "Ocupado" ou "Livre" podendo ser alterado posteriormente pelo usuário. O campo "título" deve especificar o nome do compromisso, o campo "contato" é para selecionar quais os participantes do compromisso, o campo "tipo de compromisso" deve selecionar o tipo em que pertence o compromisso, o campo "categoria de compromisso" consta qual categoria o compromisso pertence.

Para salvar um projeto, deve-se clicar no botão "Salvar", lembrando que os campos devem estar preenchidos de acordo com as instruções contidas neste guia.
